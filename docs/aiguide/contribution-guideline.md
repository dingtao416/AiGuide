# 贡献指南

感谢你对 AIGuide 项目的关注！我们欢迎任何形式的贡献。

## 🤝 如何贡献

### 1. 报告问题

如果你发现文档中的错误或有改进建议，请通过以下方式反馈：

- 在 GitHub 上提交 Issue
- 详细描述问题所在页面和具体内容
- 如有可能，提供修正建议

### 2. 提交内容

#### 新增文章

1. Fork 本仓库
2. 在对应目录下创建 Markdown 文件
3. 遵循现有文章的格式规范
4. 提交 Pull Request

#### 修改现有内容

1. 找到需要修改的文件
2. 进行必要的更正或补充
3. 确保修改后内容准确无误
4. 提交 Pull Request

## 📝 写作规范

### Frontmatter 格式

每篇文章开头需包含 YAML frontmatter：

```yaml
---
title: 文章标题
icon: 图标名称
category:
  - 分类名称
tag:
  - 标签1
  - 标签2
---
```

### 内容规范

- 使用简洁清晰的语言
- 技术术语首次出现时需解释
- 代码示例需可运行
- 适当使用图表辅助说明

### Markdown 规范

- 标题层级从 `#` 开始
- 代码块标注语言类型
- 表格对齐整齐
- 链接使用相对路径

## 🏷️ 提交规范

### Commit Message 格式

```
<type>: <description>

[optional body]
```

### Type 类型

| 类型 | 说明 |
|------|------|
| `docs` | 文档更新 |
| `fix` | 修复错误 |
| `feat` | 新增内容 |
| `style` | 格式调整 |
| `refactor` | 重构内容 |

### 示例

```
docs: 添加 RAG 技术文档

- 新增 what-is-rag.md
- 新增 embedding.md
- 更新 sidebar 配置
```

## ✅ 审核流程

1. 提交 PR 后，维护者会进行审核
2. 如有修改建议，请及时响应
3. 审核通过后会合并到主分支
4. 感谢你的贡献！

如有任何问题，欢迎通过以下方式联系：

- 项目讨论区
