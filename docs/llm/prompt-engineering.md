# Prompt 工程入门指南

## 什么是 Prompt Engineering

**Prompt Engineering（提示词工程）** 是指设计和优化输入给大语言模型的提示词（Prompt），以获得更准确、更相关、更有用的输出结果的技术和方法论。

> "Prompt 是与 AI 对话的艺术和科学"

## 为什么 Prompt 很重要

同样的模型，不同的 Prompt 可能产生天差地别的结果：

**差的 Prompt：**
```
写一篇文章
```
→ 结果：泛泛而谈，不知所云

**好的 Prompt：**
```
你是一位资深的技术博主。请写一篇关于 Python 异步编程的入门教程，
面向有1年 Python 经验的开发者。文章需要包含：
1. 什么是异步编程
2. asyncio 的基本使用
3. 实际代码示例
4. 常见陷阱和最佳实践
字数控制在 2000 字左右，语言风格轻松易懂。
```
→ 结果：结构清晰、针对性强、质量高

## Prompt 的基本结构

一个完整的 Prompt 通常包含以下要素：

```
┌─────────────────────────────────────────────┐
│               完整的 Prompt 结构             │
├─────────────────────────────────────────────┤
│  📋 角色设定（Role）                         │
│     定义 AI 应该扮演什么角色                 │
├─────────────────────────────────────────────┤
│  🎯 任务描述（Task）                         │
│     明确说明需要完成什么任务                 │
├─────────────────────────────────────────────┤
│  📝 上下文信息（Context）                    │
│     提供相关的背景信息                       │
├─────────────────────────────────────────────┤
│  📐 格式要求（Format）                       │
│     指定输出的格式和结构                     │
├─────────────────────────────────────────────┤
│  💡 示例（Examples）                         │
│     提供输入输出的示例                       │
├─────────────────────────────────────────────┤
│  ⚠️ 约束条件（Constraints）                  │
│     说明限制和注意事项                       │
└─────────────────────────────────────────────┘
```

## 核心 Prompt 技巧

### 1. 角色扮演（Role-Playing）

给模型设定一个专业角色：

```
你是一位有 10 年经验的 Python 后端工程师，
擅长系统设计和性能优化...
```

### 2. 任务分解（Task Decomposition）

将复杂任务拆分为步骤：

```
请按以下步骤完成任务：
1. 首先分析问题的核心需求
2. 然后列出可能的解决方案
3. 对比各方案的优缺点
4. 给出最终推荐方案
```

### 3. 少样本学习（Few-Shot Learning）

提供几个例子让模型学习模式：

```
请将以下英文翻译成中文，保持技术术语准确：

示例 1：
输入：Machine Learning
输出：机器学习

示例 2：
输入：Neural Network
输出：神经网络

现在请翻译：
输入：Deep Learning
输出：
```

### 4. 思维链（Chain-of-Thought, CoT）

让模型展示推理过程：

```
请一步一步思考这个问题：

问题：一个农场有 23 只羊，如果买入 6 只，卖出 10 只，最后有多少只羊？

让我们逐步分析：
```

### 5. 自洽性（Self-Consistency）

多次生成，选择最一致的答案：

```
请对这个问题给出 3 种不同的解决思路，
然后分析哪种最可靠。
```

## 实用 Prompt 模板

### 代码生成模板

```
【角色】你是一位资深的 {语言} 开发者

【任务】请编写一个 {功能描述}

【要求】
- 代码需要有详细的注释
- 包含必要的错误处理
- 遵循 {语言} 的最佳实践
- 提供使用示例

【约束】
- 不使用第三方库（或：可以使用 {库名}）
- 代码需要兼容 {版本}

【输出格式】
1. 首先解释实现思路
2. 然后给出完整代码
3. 最后提供使用示例
```

### 文本分析模板

```
请分析以下文本：
---
{待分析的文本}
---

分析维度：
1. 主题概述（50 字以内）
2. 关键观点（列出 3-5 个要点）
3. 情感倾向（正面/中性/负面）
4. 目标受众
5. 改进建议

请使用 Markdown 格式输出。
```

### 翻译优化模板

```
请将以下内容翻译成 {目标语言}：

原文：
{原文内容}

翻译要求：
1. 保持原文的语气和风格
2. 专业术语使用 {领域} 的标准译法
3. 句子流畅自然，符合 {目标语言} 的表达习惯
4. 对于难以直译的内容，给出解释说明

输出格式：
- 译文
- 翻译说明（如有）
```

## Prompt 优化技巧

### 1. 具体化

❌ `写一个函数`
✅ `写一个 Python 函数，接收一个字符串列表，返回去重后按字母排序的结果`

### 2. 结构化

使用清晰的分隔符和格式：

```
【背景信息】
...

【具体任务】
...

【输出要求】
...
```

### 3. 迭代优化

1. 从简单 Prompt 开始
2. 分析输出不满意的地方
3. 针对性地添加约束或说明
4. 重复直到满意

### 4. 设置边界

明确告诉模型什么不要做：

```
注意：
- 不要编造不存在的信息
- 如果不确定，请明确说明
- 不要使用过于复杂的技术术语
```

## 常见 Prompt 模式

### Zero-Shot（零样本）
直接描述任务，不提供示例：
```
将以下文本分类为正面或负面情感：
"这个产品太棒了！"
```

### One-Shot（单样本）
提供一个示例：
```
输入：这个产品太差了 → 负面
输入：这个产品太棒了 → ?
```

### Few-Shot（少样本）
提供多个示例：
```
输入：很失望 → 负面
输入：物超所值 → 正面
输入：一般般 → 中性
输入：强烈推荐 → ?
```

### Chain-of-Thought（思维链）
引导逐步推理：
```
Q: 15% of 80 is?
A: 15% means 15/100. 15/100 * 80 = 12. So the answer is 12.

Q: 25% of 120 is?
A: 让我们一步步计算...
```

## 高级技巧

### 1. 元提示（Meta-Prompting）
让 AI 帮你写 Prompt：
```
我想让 AI 帮我写技术文档。
请帮我设计一个效果最好的 Prompt。
```

### 2. 角色组合
模拟多个角色讨论：
```
请模拟一个产品经理和工程师讨论功能实现的对话，
最终给出可行的技术方案。
```

### 3. 反向 Prompt
告诉模型避免什么：
```
请回答这个问题，但不要：
- 使用专业术语
- 超过 100 字
- 给出模糊的回答
```

## Prompt 调试检查清单

当输出不理想时，检查：

- [ ] 角色是否清晰？
- [ ] 任务描述是否具体？
- [ ] 是否提供了足够的上下文？
- [ ] 输出格式是否明确？
- [ ] 是否有适当的示例？
- [ ] 约束条件是否合理？
